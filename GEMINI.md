# Planthor Documentation Project - Agent Instructions

This file contains foundational mandates for any AI agent (Gemini CLI) working on this project.

## Project Overview
- **Name:** Planthor Documentation
- **Framework:** [Docusaurus 3](https://docusaurus.io/)
- **Purpose:** Centralized documentation for the Planthor project and the "Dragging to Dream" (D2D) community.

## Branding & Design Mandates (CRITICAL)
- **Theme:** "Industrial Minimalist".
- **Aesthetic:** High contrast, bold borders, and **zero border-radius** on all UI elements (buttons, cards, containers).
- **Color Palette:**
  - **Primary (Navy Blue):** `#003366`
  - **Secondary/Accent (Bright Blue):** `#3399ff`
  - **Background (Light):** `#f4f4f4`
  - **Background (Dark):** `#001a33`
- **Typography:** Uppercase headings, heavy font weights (800+), and letter-spacing for an industrial feel.
- **Assets:**
  - **Logo:** Use `static/img/planthor-logo.png` for all primary branding.
  - **Favicon:** Use `static/img/favicon-32x32.png`.

## Documentation Standards
- **File Naming:** Use kebab-case for all filenames in the `docs/` directory (e.g., `c4-architecture.md`).
- **Frontmatter:** Every Markdown file in `docs/` MUST include Docusaurus frontmatter:
  ```markdown
  ---
  title: Page Title
  sidebar_label: Short Label
  sidebar_position: X
  ---
  ```
- **Entry Points:** The main documentation entry point is `docs/index.md`.
- **Diagrams:**
  - **Mermaid:** Use standard Mermaid blocks (already enabled in `docusaurus.config.ts`).
  - **PlantUML:** Since we are using MDX, PlantUML blocks MUST be wrapped in code fences to avoid curly brace parsing errors. We use `docusaurus-theme-plantuml` for rendering, which requires the `pumld` tag.
    ```pumld
    @startuml
    ...
    @enduml
    ```

## Development Workflow
- **Validation:** ALWAYS run `npm run build` after modifying documentation or configuration to ensure no broken links or build errors.
- **PlantUML Automation:**
  - If you paste a raw `@startuml` block without fences, run `npm run fix-puml` to automatically wrap it with the `pumld` tag.
- **Sidebars:** Use the `autogenerated` setting in `sidebars.ts`. Control order via `sidebar_position` in frontmatter.
- **Scripts:**
  - `npm run fix-puml`: Scans `docs/` and wraps raw PlantUML blocks in `pumld` code fences.
  - `npm run build`: Standard Docusaurus build command.
