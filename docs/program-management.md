# The Core Concept: The Triple Constraint

The triangle represents the balance between three primary forces. The "Quality" of the project is sitting right in the center, dictated by how you handle the three corners:

**Scope**: The features and functions you’re building (e.g., "It needs a calendar view and AI tagging").

**Time**: Your deadline or the hours you can commit (e.g., "We want a Beta by next month").

**Cost**: The resources involved. For a pet project, this is rarely cash—it’s usually your collective energy, snacks, and server hosting fees.

_The Golden Rule_: You cannot change one corner without affecting the others. If you want more features (Scope), you either need more time (Time) or more people/effort (Cost). If you don't adjust those, your Quality will inevitably drop.

For Planthor Program first Renovation Initiative, we focus on _**Scope**_.
Time and Cost are not entitled, but we are using as much open-source as possible.
